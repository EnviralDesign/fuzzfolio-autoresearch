## Design Context

### Users
Fuzzfolio is used by a strategy researcher assembling a temporary live portfolio from a large corpus of generated trading candidates. The main job is to shortlist many promising attempts, compare their portfolio behavior, identify redundant or highly similar strategies, and prune the set without losing sight of account risk and backtest evidence.

### Brand Personality
Precise, dense, and operational. The interface should feel like an analytical workbench: calm enough for repeated use, spatial enough to make relationships visible, and direct enough that pruning decisions are easy to make.

### Aesthetic Direction
Use the existing dark, warm-neutral dashboard palette and tight shadcn/Tailwind component language. Favor data-rich surfaces, compact controls, clear hierarchy, and interaction patterns that support scanning, comparing, and removing candidates. Avoid marketing composition, oversized hero treatment, decorative visual effects, and toy-like graph styling.

### Design Principles
- Preserve working density: the dashboard should show more candidate scorecards when portfolio controls are not actively needed.
- Make similarity spatial: portfolio sameness should be visible as clusters before the user drills into pairwise details.
- Keep pruning local: any cluster detail view should let the user remove redundant strategies without returning to the main grid.
- Keep evidence close: account simulation, backtest charts, and live set membership should remain available from the portfolio viewer.
- Prefer configurable defaults: expose a small similarity threshold control for clustering, with sensible behavior out of the box.
